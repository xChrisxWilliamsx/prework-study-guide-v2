Git

  • git status: checks what branch you are on and if there are any changes staged for commit
  • git checkout -b [branch name]: creats a new branch and switches to it.
  • git restore [file name]: disregards changes to file.
  • git add -A: adds modifications in the current working branch to the staging area. The [-A] flag indicates that we want to add all changes.
  • git commit -m [file name]: gets the items in the staging area ready to commit to GitHub.
  • git push origin [branch]/[file name]: pushes changes, as a pull request, to GitHub for main codebase.
  • git pull origin [file name from repo i.e. main]: pulls/receives a branch modifications onto local environment[your computer].
